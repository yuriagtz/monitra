# スケジュール設定とプラン機能の実装案

## 現状の確認

### 現在のスケジュール設定
- 個別LPごとにスケジュールを設定可能
- `intervalMinutes`（分単位）またはCron式で設定
- 1LPにつき1スケジュール

### 要件
1. 監視間隔は1日以上で、1日単位で設定
2. 基本的には全てのLPを監視する前提
3. プラン別の監視頻度制限：
   - フリープラン：3日に1回（最小間隔）
   - ライトプラン：3日に1回（最小間隔）
   - プロプラン：1日に1回（最小間隔）

## 実装案

### 1. ユーザーテーブルにプラン情報を追加

```typescript
// drizzle/schema.ts
export const planEnum = pgEnum("plan", ["free", "light", "pro"]);

export const users = pgTable("users", {
  // ... 既存のフィールド
  plan: planEnum("plan").default("free").notNull(), // プラン（free, light, pro）
});
```

### 2. スケジュール設定の変更

#### 案A: 全LP用の一括スケジュール設定（推奨）
- ユーザーごとに1つのスケジュール設定
- 全LPに対して同じ間隔で監視を実行

**メリット**:
- シンプルで使いやすい
- 要件「基本的には全てのLPを監視する前提」に合致
- プラン制限の実装が簡単

**データベース変更**:
```typescript
export const scheduleSettings = pgTable("schedule_settings", {
  id: serial("id").primaryKey(),
  userId: integer("user_id").notNull().unique(), // ユーザーごとに1つ
  intervalDays: integer("interval_days").notNull(), // 日単位（1, 2, 3...）
  enabled: boolean("enabled").default(true).notNull(),
  lastRunAt: timestamp("last_run_at"),
  nextRunAt: timestamp("next_run_at"),
  createdAt: timestamp("created_at").defaultNow().notNull(),
  updatedAt: timestamp("updated_at").defaultNow().$onUpdate(() => new Date()).notNull(),
});
```

#### 案B: 現状維持（個別LP設定）+ プラン制限
- 個別LP設定を維持
- プランに応じて最小間隔を制限

**デメリット**:
- 複雑になる
- 全LPを監視する前提には合わない

### 3. 監視間隔の設定方法

#### 日単位での設定
- 1日、2日、3日、7日、14日、30日などから選択
- プランに応じた最小間隔を制限

#### UI案
```
監視間隔: [ドロップダウン]
- 1日ごと（プロプランのみ）
- 2日ごと（プロプランのみ）
- 3日ごと（全プラン）
- 7日ごと（全プラン）
- 14日ごと（全プラン）
- 30日ごと（全プラン）
```

### 4. プラン制限の実装

#### プラン定義
```typescript
const PLAN_CONFIG = {
  free: {
    name: "フリープラン",
    minIntervalDays: 3, // 最小3日に1回
    maxLpCount: 5, // 最大LP登録数（将来の機能）
  },
  light: {
    name: "ライトプラン",
    minIntervalDays: 3,
    maxLpCount: 20,
  },
  pro: {
    name: "プロプラン",
    minIntervalDays: 1, // 最小1日に1回
    maxLpCount: null, // 無制限
  },
};
```

#### バリデーション
- スケジュール設定時に、プランに応じた最小間隔をチェック
- 無効な間隔を選択しようとした場合はエラーを表示

### 5. スケジュール実行の変更

現在の実装では、個別LPごとにスケジュールを実行していますが、全LP用の一括スケジュールに変更する場合：

```typescript
// server/scheduler.ts
export async function startSchedule(userId: number) {
  // ユーザーの全LPを取得
  const lps = await db.getLandingPagesByUserId(userId);
  
  // スケジュール設定を取得
  const schedule = await db.getScheduleSettingsByUserId(userId);
  
  // 全LPに対して監視を実行
  for (const lp of lps) {
    await monitorLandingPage(lp.id);
  }
}
```

## 実装の優先順位

### Phase 1: 基本機能
1. ユーザーテーブルにプラン情報を追加（デフォルト: free）
2. スケジュール設定を全LP用に変更（ユーザーごとに1設定）
3. 監視間隔を日単位で設定できるように変更
4. プランに応じた最小間隔のバリデーション

### Phase 2: UI改善
1. スケジュール設定ページのUIを更新
2. プラン情報の表示
3. プランアップグレードの案内（将来）

## データベースマイグレーション

既存の`schedule_settings`テーブルを変更する必要があります：

1. `landingPageId` → `userId`に変更
2. `intervalMinutes` → `intervalDays`に変更
3. `scheduleType`と`cronExpression`を削除（日単位のみ）

既存データの移行：
- 既存のスケジュール設定をユーザーごとに集約
- 最短間隔を新しい設定に適用
