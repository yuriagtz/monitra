# LP監視システム ユーザーガイド

## 概要

**目的**: ランディングページの変更を自動検出し、コンテンツの更新やリンク切れを監視できます

**アクセス**: ログインが必要です

## Powered by Manus

このシステムは最新のWeb技術で構築されています。

**技術スタック**:
- **フロントエンド**: React 19 + TypeScript + Tailwind CSS 4 + shadcn/ui
- **バックエンド**: Express 4 + tRPC 11 (型安全なAPI通信)
- **データベース**: MySQL (TiDB) with Drizzle ORM
- **認証**: Manus OAuth (シームレスなログイン体験)
- **スクリーンショット**: Puppeteer (Chromiumベースの自動化)
- **画像比較**: Pixelmatch (ピクセル単位の差分検出)
- **ストレージ**: S3互換オブジェクトストレージ
- **デプロイ**: 自動スケーリングインフラとグローバルCDN

## システムの使い方

### LPを登録する

「LP登録」ボタンをクリックして、監視したいランディングページのURLを入力します。タイトルと説明は任意ですが、複数のLPを管理する場合は入力しておくと便利です。「登録」をクリックすると、LPリストに追加されます。

### 手動でチェックを実行する

LPリスト画面で、各LPの右側にある更新アイコンボタンをクリックすると、即座にそのLPをチェックします。システムはスクリーンショットを撮影し、前回のスクリーンショットと比較して変更を検出します。また、URLにアクセスできるかどうかも確認します。チェックが完了すると、結果が通知されます。

### 監視履歴を確認する

各LPの目のアイコンボタンをクリックすると、そのLPの監視履歴ページに移動します。チェック日時、種類(コンテンツ変更またはリンク切れ)、ステータス、メッセージが一覧表示されます。「画像を表示」ボタンをクリックすると、現在と前回のスクリーンショット、そして差分画像をモーダルで確認できます。

### LPを削除する

不要になったLPは、ゴミ箱アイコンボタンをクリックして削除できます。確認ダイアログが表示されるので、「OK」を選択すると削除されます。

## システムの管理

### 管理UI

右側の管理パネルから以下の操作が可能です:

- **Preview**: 開発サーバーのライブプレビュー
- **Code**: ファイルツリーとダウンロード
- **Database**: データベースのCRUD操作
- **Settings**: 各種設定(環境変数、ドメイン、通知など)

### データベース

管理UIの「Database」パネルから、登録されたLP、監視履歴、スクリーンショットのデータを直接確認・編集できます。接続情報は左下の設定から確認できます(SSL有効化を推奨)。
